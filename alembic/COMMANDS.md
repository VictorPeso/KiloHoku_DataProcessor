# Alembic Commands

Run commands from the project root with the virtual environment activated:

```powershell
.\.venv\Scripts\Activate.ps1
```

If the environment is not activated, use:

```powershell
.\.venv\Scripts\python.exe -m alembic <command>
```

## Inspect Migrations

Show migration history:

```powershell
alembic history
```

Show the revision currently applied to the database:

```powershell
alembic current
```

Show generated SQL without applying it:

```powershell
alembic upgrade head --sql
```

## Apply Migrations

Apply all pending migrations:

```powershell
alembic upgrade head
```

Apply migrations up to a specific revision:

```powershell
alembic upgrade 20260626_0002
```

## Revert Migrations

Revert the last migration:

```powershell
alembic downgrade -1
```

Revert to a specific revision:

```powershell
alembic downgrade 20260626_0001
```

Revert all migrations:

```powershell
alembic downgrade base
```

## Create Migrations

Create an empty manual migration:

```powershell
alembic revision -m "describe change"
```

Create a migration from SQLAlchemy model changes:

```powershell
alembic revision --autogenerate -m "describe change"
```

Always review autogenerated migrations before applying them.

## Recommended Workflow

After changing SQLAlchemy models:

```powershell
alembic revision --autogenerate -m "describe change"
alembic upgrade head --sql
alembic upgrade head
pre-commit run --all-files
```

Do not use `Base.metadata.create_all()` for production schema creation. The
schema contract is owned by Alembic migrations.
